1. GraphQL 
: API 위한 쿼리 언어. 
 클라이언트가 필요한 데이터만 정확히 요청할 수 있음!
 기존 REST API와 달리 하나의 앤드포인트로 모든 요청 처리
 장점
  - 필요한 데이터만 요청 가능!!

 2. Apollo Server
 : GraphQL 서버를 쉽게 구축할 수 있도록 도와주는 라이브러리
  GraphQL 스키마 정의 가능, 요청 처리 로직(resolver) 구현, Express 등과 쉽게 연결 가능

2-1. typeDefs -> 스키마 정의 (요청하는 데이터의 형태 정의)
const typeDefs = `
type User {
  id: ID!
  username: String!
  age: Int
  posts: [Post]
}
type Post {
  id: ID!
  title: String!
  content: String
  author: User
}
type Query {
  users: [User]
  posts: [Post]
  user(id: ID!): User
  post(id: ID!): Post
}
`;

2-2. resolvers (실제 데이터 어떻게 반환할지 정의) / 데이터간 관계까지 직접 연결 가능
      Query -> 기본조회
      User.posts -> 유저가 쓴 글 조회
      Post.author -> 글 작성자 조회
const resolvers = {
  Query: {
    users: () => users,
    posts: () => posts,
    user: (_, { id }) => users.find((user) => user.id === id),
    post: (_, { id }) => posts.find((post) => post.id === id),
  },

  User: {
    posts: (parent) =>
      posts.filter((post) => post.authorId === parent.id),
  },

  Post: {
    author: (parent) =>
      users.find((user) => user.id === parent.authorId),
  },
};
